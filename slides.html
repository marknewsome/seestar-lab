The Challenge

What the Seestar gives you

  • Hundreds of individual .fit stacks per session
  • Typically 1–10 subs each, taken over days or weeks
  • No built-in workflow for combining comet sessions
  • Comet moves between sessions — can't simply stack

What you really want

  • Stars-fixed animation showing comet's sky motion
  • Nucleus-fixed animation showing coma evolution
  • Deep stack locked to the nucleus
  • Larson–Sekanina filter revealing jets & shells
  • A portrait image with tail oriented correctly
Seestar Lab automates the entire pipeline from raw .fit files to publication-ready animations and stacks — in a single web UI.

Smart caching

Star alignment results are saved to comet_alignment.json. Subsequent re-renders (different stretch, different crop) reuse the alignment — skipping the slowest step entirely.

Re-render without re-aligning

Adjust gamma, crop, or noise reduction and re-render in seconds. Even nucleus corrections use the cached star transforms.

Typical first render on 186 frames: ~4 minutes.
Re-render with different parameters: ~30 seconds.

Pipeline — Pass 1: Star Alignment

PASS 1every frame → reference frame similarity transform

Two-stage approach

  • Primary: astroalign — triangle-invariant hashing + RANSAC. Robust even with 30 % field overlap between sessions.
  • Fallback: OpenCV ORB features (2,000 BRIEF descriptors) with affine-partial RANSAC. Catches frames where astroalign finds too few triangles.
  • Unaligned frames are flagged (not dropped) — still usable for nucleus tracking

What the transform captures

Translation

Seestar re-points slightly between sessions

Rotation

Camera angle varies with alt/az mount

Scale

Minor plate-scale variation between nights

Pipeline — Pass 2: Nucleus Detection

PASS 2locate the comet head in every frame

The diffuseness score

Stars are sharp. Comae are fuzzy.

score = large_blur² ÷ (small_blur + ε)

σ=25 px blurs retain the coma; σ=4 px blurs retain stars. The ratio rewards extended fuzzy blobs and penalises point sources.

Sub-pixel centroid within a 40 px window around the peak reduces per-frame jitter.

User correction

  • Auto-detection can land on a nearby bright star
  • Click the actual nucleus in the frame browser → hint saved to JSON
  • Re-render uses a 120 px tight search radius around the hint — nearby stars can't steal the detection
  • Gaussian temporal smoothing (σ = 3 frames) removes remaining jitter
The hint is a fractional canvas coordinate — it works correctly even when frames are different sizes or shifted between sessions.

Output 1 — Stars-Fixed Animation

PASS 3union canvas · fill composite · blend
  • All frames warped onto a union canvas large enough to hold every aligned frame
  • Regions not covered by a given frame are filled from a mean composite of all frames — no black borders
  • Blend: pixel = frame × α + composite × (1−α)
  • VFR (variable frame rate) pacing — real timestamp gaps are preserved, so you feel cloud gaps and multi-night jumps
What you see: Stars hold rock-steady. The comet nucleus drifts steadily across the field over days or weeks — its real motion across the sky made visible.
Great for: Showing the comet's speed, direction, and any trajectory changes. Striking for audiences.

Output 2 — Nucleus-Fixed Animation

PASS 4nucleus re-centred · deep stack · trail removal
  • A composite transform applies both star-alignment and nucleus re-centring in a single warp
  • Result: nucleus locked to the same pixel in every frame; background stars trail behind it
  • Satellite and aircraft trail pixels are detected and removed before stacking
  • Sky background subtracted per-frame using only covered pixels
What you see: The comet head stays fixed. Stars draw arcs behind it. Coma structure, brightness changes, and tail development are easy to compare frame-to-frame.
Great for: Coma morphology, brightness evolution, jet activity. The deep stack from this pass rivals multi-hour dedicated imaging.

Additional Outputs

🖼 Deep Stack

Mean composite of nucleus-aligned frames. Satellite trails removed. Shows faint tail structure invisible in single frames.

🔬 Larson–Sekanina

Rotational-gradient filter (image minus rotated copy). Cancels the smooth coma gradient, revealing jets, shells, and asymmetric structures.

🎨 Portrait

Auto-cropped and auto-oriented: tail points upward, nucleus sits at lower-centre third. Comet name and date range overlaid.

📍 Track Composite

Reference frame with the smoothed nucleus trajectory drawn as a colour-graduated polyline — early frames blue, late frames red.

🎞 Frame Browser

All 186 annotated frames browsable in the UI. Click to reject cloud-out or satellite-ruined frames. Changes take effect on the next re-render.

Key controls

  • Gamma (γ): <1 boosts faint coma and tail, >1 suppresses background noise
  • Sky %: sets the background zero-point (raise for bright skies)
  • White %: controls highlight clipping (lower = brighter result)
  • Width: output pixel width (640–1920 px)
All parameters are saved between sessions — re-opening the same comet directory restores your last settings automatically.

Frame Review Workflow

What you can do

  • Browse — scroll the thumbnail strip; click any frame to view full-size
  • Reject — click a frame to flag it; rejected frames are excluded from the render but kept in the strip so you can restore them
  • Fix nucleus — click "⊕ Fix Nucleus" then click the actual comet head in the image; the hint is saved and used on the next re-render
Nucleus correction is persistent. The hint is stored in comet_alignment.json alongside the frame transforms — it survives page reloads and is automatically restored when you reopen the directory.
Re-render is fast. Hitting "Re-render →" after fixing the nucleus skips all star-alignment work and only re-runs nucleus detection and rendering — typically under a minute.

Under the Hood

🐍 Python backend

Flask · NumPy · OpenCV · astroalign · Astropy FITS · scikit-image · FFmpeg

🌐 Browser frontend

Vanilla JS · no frameworks · dark CSS design system · real-time log streaming via SSE

🗄 Persistence

SQLite for session catalogue · JSON cache per comet directory · no external services

The processor runs as a subprocess — the UI stays responsive during long renders and can poll progress, show live logs, and survive a browser refresh mid-render.

FITS compatibility

Bayer mono subs (auto-debayered) · pre-debayered stacks · BZERO/BSCALE · Seestar filename sub-count parsing

VFR video output

Frame durations proportional to real capture timestamps — a 3-day cloud gap plays slightly faster, making the time axis honest.
